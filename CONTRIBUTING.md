# Contributing to Yield Guardian

Thanks for your interest in contributing! This project is open source and welcomes contributions.

## Ways to Contribute

- **Bug Reports** ‚Äî Found something broken? Open an issue
- **Feature Requests** ‚Äî Have an idea? Start a discussion
- **Code** ‚Äî Submit a pull request
- **Documentation** ‚Äî Improve the README or add examples

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Create a branch for your changes: `git checkout -b feature/your-feature`
4. Make your changes
5. Test locally with a small amount of funds
6. Commit with clear messages
7. Push and open a pull request

## Code Guidelines

- Keep it simple ‚Äî this is meant to be understandable
- Add comments for complex logic
- Follow existing code style
- Test with real transactions (small amounts) before submitting

## Pull Request Process

1. Update the README if needed
2. Describe what your PR does and why
3. Link any related issues
4. Wait for review

## Security

If you find a security vulnerability, please **do not** open a public issue. Instead, reach out privately via GitHub's security advisory feature.
